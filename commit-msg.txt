fix: add Google OAuth2 flow for Antigravity auth (replaces broken API key)

The Interactions API requires an OAuth2 Bearer token (same Gmail account
used in the Antigravity desktop app) — a plain AIzaSy API key returns 429.

Changes:
- server/routes/googleAuth.js: new Google OAuth2 flow (/auth/google,
  /auth/google/callback, /auth/google/status, /auth/google/disconnect)
  stores access + refresh token in session; auto-refreshes on expiry
- server/services/antigravity.js: accepts googleTokens from session,
  calls getFreshAccessToken() via OAuth2Client before every request;
  removes all API-key auth code
- server/routes/chat.js: passes req.session.googleTokens to streamChat;
  returns 403 google_not_connected if Google not yet linked
- server/index.js: register googleAuthRoutes
- public/app.html: Google connect status card in sidebar
- public/js/app.js: checkGoogleStatus() on load; handles 403 gracefully
- .env: remove stale ANTIGRAVITY_API_KEY/AUTH_TYPE; add GOOGLE_CLIENT_ID,
  GOOGLE_CLIENT_SECRET, GOOGLE_CALLBACK_URL
- package.json: google-auth-library added
