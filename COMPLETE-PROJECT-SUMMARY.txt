PRIMARY BUG (User's Report):
─────────────────────────────
User: "I selected React + Go and built, but got GitHub Pages with 2 lines of HTML"

Root Cause Sequence:
  1. User selects React + Go from stack selector
  2. selectedStack saved to session: {frontend: 'react', backend: 'go', type: 'dynamic'}
  3. User clicks "Build"
  4. enrichedNotes constructed (3 possible branches):
     - Complete mode: Uses compiledSpec + stackContext ✅
     - Prototype mode: Uses planNotes + styleAnswer
     - Fallback/Edit: Uses planNotes or stackContext
  5. enrichedNotes sent to AI for code generation
  6. But in prototype mode or with missing data:
     → enrichedNotes becomes empty string!
     → AI doesn't know about Go backend
     → AI generates frontend-only GitHub Pages code
     → Deployment mode shown as "GitHub Pages" (wrong!)

Fix Applied:
  - Ensure enrichedNotes NEVER empty
  - Always include stack context if available
  - Provide fallback context if needed
  → Now React + Go builds proper Go backend + React frontend ✅

SECONDARY BUGS (Identified by Tests):
──────────────────────────────────────
Bug #2: Stack field validation
  - If stack object incomplete: buildStackContext includes 'undefined'
  - Fix: Validate all fields exist, return empty string if not

Bug #3: Deployment context
  - If stackForDeployment missing fields: wrong mode calculated
  - Fix: Verify/rebuild stack fields before deploymentMode assignment
